“单一职责原则告诉我们实现类要职责单一；
单一职责原则（Single Responsibility Principle - SRP）
里氏替换原则告诉我们不要破坏继承体系；
依赖倒置原则告诉我们要面向接口编程；
接口隔离原则告诉我们在设计接口的时候要精简单一；
迪米特法则告诉我们要降低耦合。
而开闭原则是总纲，他告诉我们要对扩展开放，对修改关闭。”

（1）借鉴优秀代码的设计，有助于提高代码设计能力
（2）JDK的设计中体现了大多数设计模式，是学习设计模式的较好的方式
（3）可以更加深入的了解JDK
