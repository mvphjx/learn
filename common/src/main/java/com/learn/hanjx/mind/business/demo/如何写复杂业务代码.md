###写复杂业务的方法论

* 自上而下的结构化分解
        
    
   在做过程分解的时候，我建议工程师不要把太多精力放在工具上，放在设计模式带来的灵活性上。
   
   而是应该多花时间在对问题分析，结构化分解，最后通过合理的抽象，形成合适的阶段（Phase）和步骤（Step）上

* 自下而上的面向对象分析

        领域模型、建模
        理清模型之间的关系
        


        
        
###业务技术&底层技术
